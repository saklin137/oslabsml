Zombie Process:

A process which has finished the execution but still has entry in the process table to report to its parent process is known as a zombie process. A child process always first becomes a zombie before being removed from the process table. The parent process reads the exit status of the child process which reaps off the child process entry from the process table.

Below is an example of code to create a zombie process

  • In C sleep() function is defined under “unistd.h”
  • sleep() function return integer

#include<stdio.h>
#include<unistd.h>
#include<stdlib.h>
int main()
{
int childid = fork();
if(childid==0){
printf("\n\n");
system("ps u");
printf("\n\n");
}else{
printf("\n\n");
system("ps u");
printf("\n\n");
//parent process sleep for 5 seconds
sleep(5);
printf("\n\n");
system("ps u");
printf("\n\n");
}
return 0;
}
